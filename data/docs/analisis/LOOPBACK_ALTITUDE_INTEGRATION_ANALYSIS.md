## When & How to Integrate Proprioceptive Feedback with the Brain

**Date:** 2026-03-13  
**Status:** Analysis & Recommendations  
**Priority:** High - Critical for physics simulation

## CURRENT BRAIN STRUCTURE

The ImprovedOlfactoryBrain currently receives:
- **Position (x, y, z)** → Only x, y used; z ignored
- **Heading (radians)** → Body orientation
- **Sensory input** → Olfactory concentration readings

The brain outputs:
- **forward** (0.0 to 1.0) → Walking speed command
- **turn** (-1.0 to +1.0) → Turning intensity & direction

**Problem:** No altitude (z) awareness, no feedback from physics simulation

---

## THE LOOPBACK PROBLEM

### What the fly needs to sense:

1. **Altitude awareness:**
   - Does the fly need to climb/descend?
   - Is it falling (z velocity < 0)?
   - Is it moving away from surface (z > max height)?

2. **Leg feedback (proprioception):**
   - Are legs contacting ground?
   - What force are legs exerting?
   - What is the total load on legs?

3. **Physics feedback:**
   - Actual vs commanded velocity mismatch?
   - Directional slip (turning too fast)?
   - Stability measures?

4. **Body state:**
   - Current pitch/roll angles?
   - Walking direction vs heading?
   - Actual speed vs commanded speed?

### Current state of simulation:

In `run_complete_3d_simulation.py` (outputs/simulations):
- Motor commands: `(forward, turn)` generated by brain
- CPG processes these into leg movements
- Physics simulates leg-ground interaction
- Fly falls due to gravity if legs don't push hard enough
- ⚠️ **NO FEEDBACK TO BRAIN** - Brain never learns the fall is happening

---

## WHAT NEEDS TO CHANGE?

### Architecture Options:

```
OPTION A: Brain Processes Loopback (Complex)
═══════════════════════════════════════════════

Sensorimotor loop:
  Position ──────> Brain ──────> (forward, turn) ──> CPG
    ^                                                   ↓
    │                                                  Physics
    │                                                   ↓
    └───── [Physics feedback loop] <─────────────────────┘

Pro:  Brain learns altitude, adapts behavior
Con:  Brain becomes stateful, complex feedback integration
Risk: Stability issues if feedback causes oscillations


OPTION B: Middleware (CPG) Processes Feedback (Recommended)
═════════════════════════════════════════════════════════════

Sensorimotor loop:
  Position ──────> Brain ──────> (forward, turn) ──────> CPG ──────> Physics
    ^                                                       ↓
    │                                                   Feedback Layer
    │                                  [Altitude control, traction check]
    │                                                       ↓
    └─────────────────────────────────────────────────────────┘

Pro:  Central control point, modular separation
      Brain stays simple & fast
      CPG handles altitude autonomously
Con:  CPG becomes more complex (but it should be)


OPTION C: Separate Altitude/Stability Controller
════════════════════════════════════════════════════

Three-layer system:
  Olfactory Brain (forward, turn) ──┐
  Altitude Controller       (climb) ──> CPG ──> Physics
  Stability Controller      (pitch) ──┘

Pro:  Clean separation of concerns
      Each system independently tunable
Con:  More code, coordination needed


OPTION D: Hybrid - Brain gets Minimal Altitude Info
═══════════════════════════════════════════════════════

Simple feedback:
  Position ──────> Brain ──────> (forward, turn) ──> CPG
    ^                             ↑
    │                             │
    └─────── [Altitude flag] ─────┘
             (on_ground: bool)
             (falling: bool)

Pro:  Brain stays simple, gets critical info
      No complex feedback loops
Con:  Still requires new input to brain


```

---

## RECOMMENDED APPROACH: OPTION B (CPG Middleware)

### Why Option B is best:

1. **Olfactory brain stays focused**
   - Only processes odor information
   - Outputs simple (forward, turn) commands
   - No altitude complications

2. **CPG becomes the sensorimotor integration layer**
   - Takes brain commands as input
   - Reads physics feedback
   - Adjusts leg timings for altitude/traction
   - Outputs leg forces

3. **Clear modularity**
   - Brain = Behavior (what to do)
   - CPG = Motor control (how to do it)
   - Physics = Simulation (what happens)

4. **Biological plausibility**
   - Real Drosophila have similar:
     - Higher brain centers (mushroom bodies) → behavior decisions  
     - Central pattern generators → motor execution
     - Sensory feedback to CPG → online adaptation

## IMPLEMENTATION PLAN

### Phase 1: CPG Altitude Control (Weeks 1-2)

**Objective:** Enable fly to maintain ground contact

**Changes to `src/controllers/cpg_controller.py`:**

```python
class CPGController:
    def __init__(self, ...): ...
    
    def update_with_feedback(self,
                            motor_commands: tuple,      # (forward, turn) from brain
                            current_position: ndarray,   # (x, y, z)
                            leg_forces: dict,            # {'leg_id': force_vector}
                            contact_state: dict,         # {'leg_id': bool}
                            )
                            -> leg_amplitudes:
        """
        Process brain commands and physics feedback.
        
        Inputs:
          - forward/turn: What the brain wants
          - current_position: Where fly is now
          - leg_forces: What legs are experiencing
          - contact_state: Which legs touch ground
          
        Outputs:
          - Modified leg amplitudes accounting for altitude
        """
        
        # Calculate altitude error
        z_target = 2.0  # mm (desired height above ground)
        z_error = z_target - current_position[2]
        
        # Altitude correction (PID-like)
        # If falling: increase leg push
        # If climbing: reduce leg push
        altitude_correction = proportional_gain * z_error + derivative_gain * z_velocity
        
        # Check traction
        legs_in_contact = sum(contact_state.values())
        if legs_in_contact < 3:  # Need 3+ legs for stability
            # Reduce walking speed, increase stance phase
            forward *= (legs_in_contact / 6.0)  # Scale down by contact fraction
        
        # Apply corrections to CPG amplitudes
        adjusted_amplitudes = self._apply_feedback_to_cpg(
            motor_commands=motor_commands,
            altitude_correction=altitude_correction,
            contact_state=contact_state
        )
        
        return adjusted_amplitudes
```

**Integration with simulation loop:**

**What to add to loopback:**

```python
ProprioceptiveFeedback = {
    'altitude': {
        'z_position': float,              # Current z

        'ground_contact': bool,           # Any legs touching
        'contact_count': int,             # How many legs touch

    'traction': {
        'slip_left': float,               # -1 to +1 (slip amount)

        'total_grip': float,               # 0-1 (how well gripping)
    },
    'stability': {
        'pitch_angle': float,              # Body rotation
        'roll_angle': float,               # Side tilt
        'wanted_vs_actual_heading': float, # Navigation error
        'forward_velocity': float,         # Measured vs commanded
    }
}
```

**Where to apply this feedback:**

- **Altitude**: CPG → increase leg force if falling
- **Traction**: CPG → reduce turning if slipping
- **Stability**: Optionally → Brain for behavioral adaptation (e.g., stop odor search if unstable)

### Phase 3: Brain Adaptation to Feedback (Optional, Weeks 5+)

**Only if needed after Phase 1-2 work:**

```python
class AdaptiveOlfactoryBrain(ImprovedOlfactoryBrain):
    """Brain that responds to critical feedback signals."""
    
    def __init__(self, ...):
        super().__init__(...)
        self.stability_mode = False  # Flag if fly is unstable
        
    def step(self, odor_field, position, heading, **feedback):
        """
        Args:
            feedback: dict with 'ground_contact', 'pitch_angle', etc.
        """
        # Get normal olfactory drive
        forward, turn = super().step(odor_field, position, heading)
        
        # Modulate by stability (optional)
        if feedback.get('ground_contact', True) == False:
            # Flying in air - different behavior needed
            forward *= 0.5  # Reduce walking drive
            self.stability_mode = True
        else:
            self.stability_mode = False
        
        return forward, turn
```
